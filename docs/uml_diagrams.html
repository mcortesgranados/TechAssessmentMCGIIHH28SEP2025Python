20 UML Diagrams

Structural Diagrams

  • 📦 Class Diagram: Models classes, attributes, methods, and relationships between classes.
  • 🧩 Object Diagram: Represents instances of classes and their relationships at a specific moment.
  • 🖥️ Component Diagram: Shows software components and their dependencies.
  • 🛠️ Deployment Diagram: Represents physical nodes, devices, and artifact deployment.
  • 📁 Package Diagram: Organizes classes and components into packages for modularity.
  • 🔧 Profile Diagram: Extends UML with stereotypes and custom annotations.
  • 🏗️ Composite Structure Diagram: Describes the internal structure of classes and components with part relationships.

Behavioral Diagrams

  • 👤 Use Case Diagram: Identifies actors and system functionalities.
  • 🏃 Activity Diagram: Models workflows or processes inside the system.
  • 🔄 State Diagram: Describes object states and transitions between them.
  • ⏱️ Sequence Diagram: Shows interaction between objects over time.
  • 💬 Communication Diagram: Focuses on messages exchanged between objects or components.
  • ⏳ Timing Diagram: Models changes in objects with respect to time.
  • 🔗 Interaction Overview Diagram: Combines sequence and activity flows to describe interactions.

Additional / Extended Diagrams

  • 📊 Data Flow Diagram: Represents flow of information between processes (practical extension).
  • 🏢 Business Process Diagram: Visualizes business processes and decision points.
  • 🗃️ Entity-Relationship Diagram: Models database structures and entity relationships.
  • 🏗️ Software Architecture Diagram: Represents high-level components and system connections.
  • 🔄 Extended State Transition Diagram: Describes complex object behaviors and detailed states.
  • 🔌 Interface / API Diagram: Documents how components interact at the interface or contract level.
Tip: Use these diagrams to comprehensively document both the structure and behavior of your system.
